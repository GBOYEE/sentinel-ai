# Security Policy

## Supported Versions

Only the latest `main` branch receives security updates.

## Reporting a Vulnerability

If you discover a security issue, please **do not** open a public issue. Email the maintainer directly:

**Email:** Oyebanjiadegboyee@gmail.com

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We aim to respond within 48 hours and will work with you to resolve the issue. Once fixed, we will publicly acknowledge your contribution (if desired) in the release notes.

## Best Practices for Deployers

- Keep GitHub App private key secure and rotate periodically
- Use strong webhook secret
- Restrict the GitHub App to specific repositories only
- Monitor logs for suspicious activity
- Keep dependencies updated (dependabot enabled)
